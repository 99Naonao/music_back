# 宝塔 · 替换默认 404 页面

浏览器里出现 **「Power by 堡塔」** 的 404，是 **Nginx 静态层**返回的，不是 Node 程序。可按下面步骤换成项目定制页。

## 1. 上传定制页

将仓库中的文件复制到站点根目录（与 `data` 同级）：

```text
/www/wwwroot/music_sleep_api/404.html
```

源文件路径：`backend/deploy/public/404.html`

## 2. 修改站点 Nginx 配置

宝塔 → **网站** → 你的站点（`music.zsyl.cc`）→ **配置文件**，在 `server { ... }` 内、`location /` 之前或之后增加：

```nginx
# 自定义 404（路径为站点根目录下的 404.html）
error_page 404 /404.html;
location = /404.html {
    root /www/wwwroot/music_sleep_api;
    internal;
}
```

**注意：** 把 `root` 改成你真实的项目根路径。

保存后 **重载 Nginx**（宝塔会提示重载配置）。

## 3. 仍出现堡塔 404 时

说明请求**没有进 Node**，常见原因：

| 现象 | 处理 |
|------|------|
| 访问 `*.mp3` / `*.png` 直链 | 改用 API：`/api/music/library-audio?f=track1.mp3` |
| `/api/` 接口也是堡塔 404 | 在配置里加 `location ^~ /api/` 反代到 Node（见 `nginx-upload-cover.conf.example`） |
| 改完配置无效 | 检查是否有「静态文件优先」规则盖住了 `error_page` |

推荐 API 反代片段（与定制 404 可同时存在）：

```nginx
location ^~ /api/ {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

location ^~ /audio/ {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

端口 `3000` 改成你宝塔 Node 项目实际端口。

## 4. 验证

- 故意访问一个不存在的路径，例如：`https://music.zsyl.cc/this-path-not-exist`  
  应看到 **「这段旋律走丢了」** 定制页，而不是堡塔绿链页。
- API 正常：`https://music.zsyl.cc/api/music/library?limit=1` 应返回 JSON。

## 5. 修改文案 / 样式

直接编辑服务器上的 `404.html`，改完无需重启 Node，刷新浏览器即可（可强刷 Ctrl+F5）。
