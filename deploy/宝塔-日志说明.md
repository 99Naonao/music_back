# 宝塔部署 · 按天日志

## 1. 环境变量（`.env.production` 或宝塔 Node 项目环境变量）

```env
LOG_DIR=/www/wwwroot/你的项目目录/logs
LOG_RETAIN_DAYS=30
LOG_CONSOLE=1
```

- `LOG_DIR`：填项目 **`logs` 根目录**（绝对路径）。程序会自动写到其下的 **`log/` 子目录**。
- `LOG_FLAT=1`：可选；设为 `1` 时按天日志直接写在 `LOG_DIR` 根目录（旧行为，一般不推荐）。
- `LOG_RETAIN_DAYS`：保留天数，启动时自动删除更早的 `app-*.log` / `error-*.log`。
- `LOG_CONSOLE=1`：同时输出到控制台（pm2/宝塔面板里也能看到）；仅写文件可设为 `0`。

## 2. 目录结构说明

| 路径 | 来源 | 说明 |
|------|------|------|
| `logs/` 根目录下少数固定文件 | 宝塔 Node / pm2 | 进程 stdout/stderr 等，**不是**业务按天日志 |
| `logs/log/app-YYYY-MM-DD.log` | 本后端 `logger.js` | 当天全量业务日志（JSON 行） |
| `logs/log/error-YYYY-MM-DD.log` | 本后端 `logger.js` | 当天仅 ERROR |

## 3. 按天日志文件内容

| 文件 | 内容 |
|------|------|
| `app-2026-05-19.log` | 当天全部 INFO / WARN / ERROR（一行一条 JSON） |
| `error-2026-05-19.log` | 当天仅 ERROR |

HTTP 请求在 `[HTTP]` 上下文，示例：

```json
{"time":"2026-05-19 14:30:01.234","level":"INFO","context":"HTTP","message":"GET /api/promo/active 200 12ms","method":"GET","path":"/api/promo/active","status":200,"ms":12,"ip":"1.2.3.4","bizCode":0}
```

## 4. 宝塔面板查看

1. 文件 → 进入 **`logs/log/`**（例如 `/www/wwwroot/music_sleep_api/logs/log/`），按日期打开 `app-YYYY-MM-DD.log`。
2. 或用 SSH：`tail -f /www/wwwroot/music_sleep_api/logs/log/app-$(TZ=Asia/Shanghai date +%Y-%m-%d).log`

## 5. 目录权限

确保运行 Node 的用户对该目录可写：

```bash
mkdir -p /www/wwwroot/music_sleep_api/logs/log
chown -R www:www /www/wwwroot/music_sleep_api/logs
chmod 755 /www/wwwroot/music_sleep_api/logs
```

（用户可能是 `www` 或你宝塔 Node 项目指定的用户。）

## 6. 重启生效

修改 `LOG_DIR` 后需重启 Node 项目（宝塔「Node 项目」→ 重启，或 `pm2 restart`）。启动后 **out.log / combined.log** 里应出现：

```text
[INFO] [logger] 日志目录 fileLog=true { logDir: '/www/wwwroot/music_sleep_api/logs/log', ... }
```

若 `fileLog=false` 或 `[ERROR] [logger] 无法写入日志目录`，说明 **logs/log 目录权限不对**（见下节）。

## 6.1 自检命令（SSH / 宝塔终端）

```bash
cd /www/wwwroot/music_sleep_api
node scripts/log-self-test.js
```

成功会打印 `OK 已生成 .../logs/log/app-YYYY-MM-DD.log`。

## 6.2 只有 out.log / err.log / combined.log，没有 app-*.log？

| 原因 | 处理 |
|------|------|
| 未部署含 `logger.js` 新代码 | 上传后端并重启 |
| 宝塔未设 `NODE_ENV=production` 且未读 `.env.production` | 新版已自动补读；或在 Node 项目环境变量加 `LOG_DIR=/www/wwwroot/music_sleep_api/logs` |
| `logs/log` 属主不是运行 Node 的用户 | `chown -R www:www /www/wwwroot/music_sleep_api/logs` |
| 旧版代码把日志写在 `logs/` 根目录 | 升级后看 `logs/log/`；或设 `LOG_FLAT=1` 保持根目录（不推荐） |

## 7. 与宝塔自带日志的关系

- **`logs/` 根目录** 里宝塔/pm2 的少量固定文件可保留，不必删。
- **业务排查** 请看 **`logs/log/`** 下 `app-*.log` / `error-*.log`。
- `LOG_CONSOLE=1` 时 pm2 面板里也能看到与文件一致的输出。

## 8. 日志出现在 `music_slepp_api`（拼写错误目录）？

`wwwroot` 下若同时存在 **`music_slepp_api`**（slepp）和 **`music_sleep_api`**（sleep），说明曾建过两个目录。  
**日志写在哪个目录，取决于 Node 实际从哪个目录启动**，与你在文件管理里浏览的目录可能不是同一个。

| 目录 | 常见情况 |
|------|----------|
| `music_slepp_api` | 宝塔 Node 项目「项目路径 / 运行目录」指到了这里；属主常为 `root` |
| `music_sleep_api` | 你上传 mp3、`.env.production`、曲库的正确目录；属主常为 `www` |

**处理：**

1. 宝塔 → **Node 项目** → 打开当前在跑的项目 → 看 **项目路径 / 根目录 / 启动文件** 必须是：  
   `/www/wwwroot/music_sleep_api`（不要带 `slepp`）
2. 环境变量里的路径全部用 **`music_sleep_api`**：  
   `LOG_DIR=/www/wwwroot/music_sleep_api/logs`  
   `DB_PATH=/www/wwwroot/music_sleep_api/data/sleep_music_v2.db`
3. **重启** Node 后，在 `music_sleep_api/logs/log/` 看 `app-*.log`；`slepp` 目录可确认无进程后再删或改名备份。
4. 本地仓库与 `.env.production` 中只有 **`music_sleep_api`**，没有 `slepp`；若服务器仍写到 slepp，一定是面板/PM2 路径未改。
